# Contributing to Travel Analysis Skillset

Thank you for your interest in contributing! This project aims to encode travel demand modeling expertise as AI skills for transportation planners.

## How to Contribute

### Knowledge Contributions

The most valuable contributions are improvements to the skill content:

1. **Correct inaccuracies** - If you spot something wrong, open an issue or PR
2. **Fill gaps** - If an important topic is missing or underexplained, add it
3. **Add references** - Link to authoritative sources, especially tfresource.org pages
4. **Improve practical guidance** - Add common pitfalls, rules of thumb, or practitioner tips

### Guidelines

- **Be practical**: Target working planners, not academic audiences
- **Reference sources**: Include links to tfresource.org, NCHRP reports, or other authoritative references
- **Keep it concise**: The skill should be scannable. Use short, direct explanations.
- **Test your changes**: Try invoking the skill in Claude Code to make sure it produces useful responses

### Skill File Format

Each skill is a `SKILL.md` file with YAML frontmatter:

```markdown
---
name: skill-name
description: One-line description
trigger: when-to-activate
---

Skill content here...
```

## Code of Conduct

Be respectful, constructive, and focused on improving the resource for practitioners.
